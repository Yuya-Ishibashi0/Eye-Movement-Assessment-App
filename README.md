# 子どもの眼球運動評価アプリ (Eye Movement Assessment App)

## 概要

本アプリは、標準化された眼球運動課題（PowerPointベースの内容）をブラウザ上で再現するWebアプリケーションのMVP（Minimum Viable Product）です。診断用ではなく、子どもの眼球運動の特徴を同一条件下で記録・後から比較・評価するための補助ツールとして設計されています。
専用のアイトラッカーを必要とせず、PCまたはタブレットの内蔵カメラのみで動作します。

## 今回のMVPでできること / できないこと

### 今回のMVPでできること
- **標準化された課題刺激の提示**: 注視、左右サッケード、上下サッケード、左右追視、上下追視、円追視の6種類のプロトコルを画面上に自動/手動で提示します。
- **画面キャリブレーション**: クレジットカードや定規などの物理的な基準を用いて、画面上の表示サイズ（mm/cm）を簡易的に補正し、物理サイズを再現します。
- **カメラ録画機能**: ブラウザと内蔵の前面カメラを使用し、各課題ごとの顔の挙動を録画します。サーバーへのアップロード等はなく、ローカルに動画として保存・ZIP書き出しが可能です。
- **簡易評価サポートUI**: 検査者が手動で観察結果をメモし、簡易スコアを付けられるユーザーインターフェースが備わっています。

### 今回のMVPでできないこと
- **専用アイトラッカー並みの高精度な視線追随の完全自動解析**。
- アプリによる医療的診断、または絶対的な「正常・異常」の自動判定。
- バックエンドサーバーやクラウド環境での患者データの共有、セッション間の直接的な比較や履歴管理（将来拡張予定）。
- 輻輳テストなどのより複雑な眼球運動機能のテスト。

## 機能一覧
- **セットアップ機能**: 被験者ID等のメタデータ入力、画面キャリブレーション、視距離ガイド・被験者との位置合わせUI（プレビュー）。
- **検査実行機能**: 全画面表示での課題提示、途中停止・再開、子ども向けシンプルUI（ノイズレス設計）、検査者向け管理UI（残り時間、次課題の表示）。
- **カメラ録画と保存機能**: 対象課題ごとのブラウザ完結での動画録画と、ローカルストレージ・ファイルダウンロード対応。
- **簡易分析基盤**: 頭部挙動の簡易推定やイベントログ保存とCSV/JSON出力（将来的にMediaPipeの導入と連動）。

## 画面一覧
1. **セットアップ画面**
   - 被験者情報、年齢、検査者名、実施日の入力。
   - 画面サイズ補正機能。
   - 顔がカメラに収まっているかのプレビュー画面。
2. **検査実行画面**
   - シンプルな刺激提示（黒い点、赤色・青色の図形等）。
   - 課題名や残り時間を小さく表示する検査者向けUI。
   - 中央の刺激に集中させるためのノイズのないレイアウト。
3. **結果画面**
   - 課題ごとの録画プレビューおよび実施可否の表示。
   - 観察メモの入力フィールド・再測定フラグのチェック。
   - ダウンロード（エクスポート）機能および結果の全体サマリー（課題ごとの一覧表）。

## 将来の精度改善・機能拡張案
- **MediaPipe等による頭部・視線の推論強化**: 顔ランドマーク等を用いた頭部運動の可視化と簡易スコア化（NSUCOなど）。目領域からの視線推論。
- **セッション履歴とクラウド保存**: トレーニング前後での追跡機能、クラウドデータベースを通した履歴管理。
- **自動レポート生成**: 施設内および保護者に向けた簡単なPDFレポートの自動出力。
- **検査者間比較機能と施設向けダッシュボード**。

## 医療用途での注意点
**【重要】本アプリは医療診断ツール（医療機器）ではありません。**
- 本アプリが提示する数値や推論結果（今後導入されるスコアや顔・視線推論等含む）はあくまで「参考値」です。
- アプリの結果のみを用いて確定診断や過度な医療的解釈を行わないでください。特徴の把握と検査後の観察・比較に特化しています。
- 推定精度にはブラウザおよびWebカメラという環境上の限界があることをご理解の上、現場の所見を優先してください。

## 実装上の難所と回避策
1. **ブラウザやWebカメラのみでの高い精度の眼球運動推定の難しさ**
   - アイトラッカーがないため、瞳孔の位置や微小な動きを子どもに対して安定して取得することはノイズが多く困難です。
   - **回避策**: 自動判定や推論に完全に依存するのではなく、「課題刺激の標準化と動画の録画・表示」という基盤となる部分をMVPの最優先機能にしています。検査者が動画を見返して手動で評価できる堅牢なワークフローを構築しました。
2. **大容量の動画データのローカル保存の制約**
   - ブラウザ上で大量の動画ファイル（複数の課題分）を保持し続けるとメモリを圧迫します。
   - **回避策**: 録画データを課題ごとにBlobとして軽量な形でローカル保持させ、測定終了後にZIPなどの形式で速やかにローカル出力できる仕組みを導入する構成としています。

## 開発環境と起動方法

本プロジェクトは `React` + `TypeScript` + `Vite`、UIには `Tailwind CSS` を使用して構築されています。

### 前提条件
- Node.js (v18以降推奨) がインストールされていること

### セットアップ
```bash
npm install
```

### 開発サーバー起動
```bash
npm run dev
```
