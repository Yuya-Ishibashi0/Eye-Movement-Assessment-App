# **指示書**

**あなたは、子どもの眼球運動を簡便に評価するWebアプリを設計・実装するプロダクトエンジニアです。**
PowerPointベースで標準化された眼球運動課題を、ブラウザで再現するWebアプリのMVPを作成してください。

## 目的

* 検査者の主観に依存しやすい眼球運動評価を、提示条件をそろえたWebアプリで再現する
* 医療診断を行うためのアプリではなく、**子どもの眼球運動の特徴を同条件で記録・比較する評価補助ツール**として使う
* 専用アイトラッカーは使わず、**PCまたはタブレットの内蔵カメラのみ**で動作する
* まずは**刺激提示の標準化・録画・簡易分析・結果保存**までをMVPとする

## 参考にすべき評価課題

以下の課題を実装してください。
物理サイズをできるだけ再現したいので、アプリ起動時に**画面キャリブレーション機能**を入れてください。
たとえば、クレジットカード幅や定規を使って、画面上のmm/cmを補正できるようにしてください。

### 1. 注視

* 画面中央に**黒い5mmの点**
* **10秒間**表示
* 子どもは中央の点を見続ける

### 2. 左右サッケード

* 画面中央から**左9cm**に赤い丸、**右9cm**に青い四角
* それぞれの大きさは**5mm**
* 左右を**交互に5往復**
* 実装は、検査者が手動で「丸」「四角」を指示してもよいし、音声や画面ガイドで自動化してもよい
* MVPでは**固定順序で交互**でもよい

### 3. 上下サッケード

* 画面中央から**上9cm**に赤い丸、**下9cm**に青い四角
* それぞれの大きさは**5mm**
* 上下を**交互に5往復**
* 左右サッケードと同様に、手動指示または自動ガイド対応

### 4. 左右追視

* **黒い5mmの点**が、中央から**左9cm**の位置から**右9cm**まで移動
* **速度は5.652cm/秒**
* 端点で**1秒停止**
* **2往復**
* 子どもは点を目で追い続ける

### 5. 上下追視

* **黒い5mmの点**が、中央から**上9cm**の位置から**下9cm**まで移動
* **速度は5.652cm/秒**
* 端点で**1秒停止**
* **2往復**
* 子どもは点を目で追い続ける

### 6. 円追視

* **直径18cm**の円軌道上を、**黒い5mmの点**が移動
* 開始位置は**円の上端**
* **1周目と3周目は時計回り**
* **2周目と4周目は反時計回り**
* **速度は5.652cm/秒**
* 子どもは点を目で追い続ける

### 休憩

* 各課題の間に**約10秒の休憩画面**を入れる

## MVPで必須の機能

### A. セットアップ画面

* 被験者ID入力
* 年齢入力
* 検査者名入力
* 実施日
* 使用端末
* 画面サイズキャリブレーション
* 視距離ガイド表示
* 画面中心を被験者の目の高さに合わせるためのセットアップ案内
* 顔がカメラに収まっているかのプレビュー表示

### B. 検査実行画面

* 全画面表示
* 子ども向けにシンプルでノイズの少ないUI
* 検査者向けには、現在の課題名、残り時間、次の課題を小さく確認できる管理UI
* 各課題開始前に短い説明画面
* スペースキーまたはボタンで開始
* 途中中断、一時停止、再開ができる

### C. カメラ録画

* 各課題中、前面カメラで顔動画を録画
* 動画は課題ごとに保存
* 可能ならブラウザ内だけで完結し、サーバー送信なしでも使えるようにする
* ローカル保存またはZIPエクスポート対応

### D. 簡易分析

専用アイトラッカーがないため、**ブラウザで可能な範囲の推定**に留めてください。
精度を誇張しないこと。

優先順位は以下です。

1. **頭部運動の推定**

* MediaPipe Face Mesh などで顔ランドマークを取得
* 頭部の回旋・上下動・左右移動を推定
* 課題中の頭の動きの大きさを可視化
* NSUCO風に、**頭の動き 1〜5** の簡易スコアを出す
* ただし自動判定だけに依存せず、**検査者が後から手動修正できるUI**も用意する

2. **視線追従の簡易推定**

* 可能であれば、目領域や虹彩ランドマークから、ターゲット方向への追従を簡易推定する
* ただしノイズが大きい場合は、無理に厳密スコア化しない
* 信頼度が低い場合は「参考値」または「再測定推奨」と表示する
* 絶対的な正常/異常判定はしない

3. **イベントログ**

* 刺激の開始時刻
* 方向転換時刻
* 休憩開始/終了
* 録画開始/終了
* 推定値の時系列
* これらを JSON / CSV で出力

### E. 結果画面

課題ごとに以下を表示してください。

* 課題名
* 実施可否
* 録画プレビュー
* 頭部運動グラフ
* 簡易スコア
* 観察メモ
* 再測定フラグ
* エクスポートボタン

全体サマリーとして、

* 注視
* 左右サッケード
* 上下サッケード
* 左右追視
* 上下追視
* 円追視
  を一覧で見られるようにしてください。

## 重要な設計方針

* **診断アプリにしない**
* **特徴把握と経時比較のための評価補助**に徹する
* 数値は「参考値」として扱い、過度な医療的解釈をしない
* 結果は「今回の測定条件下での傾向」として表示する
* 検査者の観察所見を必ず残せるようにする
* 自動解析が不安定でも、**刺激提示と録画だけで使える設計**にする
* つまり、MVPとしては「標準化された刺激提示＋録画＋レビュー」だけでも成立させる

## 技術要件

* React + TypeScript
* Vite もしくは Next.js のどちらかでよいが、**軽くてローカル実行しやすい構成**を優先
* Tailwind CSS でUI実装
* カメラアクセスは WebRTC / MediaDevices API
* 顔推定は MediaPipe 系を優先
* まずは**フロントエンド完結**を優先し、バックエンドは不要なら作らない
* PWA化できるなら望ましい
* データはローカル保存を基本にする

## UIトーン

* 子ども向けだが幼すぎない
* 余計な装飾は少なく、検査刺激が見やすい
* 検査者向けUIは落ち着いた医療・教育寄り
* 被験者向け画面は中央の刺激に集中しやすいデザイン
* 音やアニメーションは最小限

## 最後に

最初から完璧な視線推定を目指さず、
**「PowerPointでやっていた標準化された刺激提示を、ブラウザで安定再現すること」** を最重要目標にしてください。
その上で、**録画・頭部運動・簡易な眼球追従推定**を段階的に加えてください。
また、**推定精度に限界がある点をUIとREADMEに明記**してください。
